# workspace-profile-menu Specification

## Purpose

工作台资料名片为个人入口单源：菜单顺序、登录更多账号、设置入口、会话头精简。

## Requirements

### Requirement: 工作台资料名片为个人入口单源

工作台 Rail 点击当前用户头像后，系统 MUST 展示资料名片；名片头部 MUST 将头像与昵称排在同一水平轴（头像左、昵称与企业名右）。名片 MUST 提供以下菜单顺序：个性签名（占位）、我的个人名片（占位）、登录更多账号、设置、退出登录；管理后台入口（若当前用户具备管理身份）MUST 置于最底部独立分区。

#### Scenario: 打开名片看到同轴头部与菜单

- **WHEN** 已登录成员在工作台点击 Rail 头像
- **THEN** 弹出资料名片，头像与昵称同一水平轴
- **AND** 可见个性签名与我的个人名片的显式占位项
- **AND** 可见登录更多账号、设置、退出登录
- **AND** 若 `isAdmin` 为真，最底部可见管理后台入口

#### Scenario: 已接通能力仍可用

- **WHEN** 成员在名片中编辑昵称或更换头像
- **THEN** 行为与现有 `/app-api/system/user/profile` 读写一致
- **WHEN** 成员点击退出登录
- **THEN** 执行现有退出流程（可保留本机其他 Account Dock 会话）
- **WHEN** 管理身份成员点击管理后台
- **THEN** 进入 `/admin`

#### Scenario: 占位项可识别且有反馈

- **WHEN** 成员点击标为占位的菜单项（个性签名、我的个人名片等）
- **THEN** UI 明确标识其为占位（文案或标签）
- **AND** 给予可见反馈（如 toast「功能即将推出」），不得静默无响应

### Requirement: 登录更多账号面板列出可切换企业与会话

系统 MUST 在名片「登录更多账号」下提供二级面板。面板 MUST 展示当前身份已绑定的其他企业（来自租户列表，排除当前租户）以及本机已登录的其他账号会话（Account Dock，排除当前会话），并支持一键切换。面板下部 MUST 提供「加入企业」显式占位与「创建新账号」入口。

#### Scenario: 列出并切换其他企业

- **WHEN** 当前用户存在多个 ACTIVE 企业成员关系且打开登录更多账号面板
- **THEN** 列表展示非当前企业的项（含企业名与展示名）
- **AND** 选择一项后完成租户切换并刷新工作台会话上下文

#### Scenario: 列出并切换本机其他登录会话

- **WHEN** Account Dock 中存在非当前 `userId:tenantId` 会话且打开该面板
- **THEN** 列表展示这些会话
- **AND** 选择一项后切换到对应会话（沿用现有 dock 切换逻辑）

#### Scenario: 底部占位与创建入口

- **WHEN** 成员查看登录更多账号面板底部
- **THEN** 可见「加入企业」且标明占位
- **AND** 可见「创建新账号」入口（可导航至现有注册/加号登录流程）

### Requirement: 设置含主题切换且主题不浮在壳层右上角

工作台已登录壳层 MUST NOT 在右上角常驻主题开关。成员 MUST 能从资料名片进入设置，并在设置中切换浅色/深色主题。未登录认证页可保留独立主题开关。

#### Scenario: 从名片进入设置切换主题

- **WHEN** 成员在资料名片点击设置
- **THEN** 打开设置面板且可见主题切换控件
- **WHEN** 成员切换主题
- **THEN** 工作台立即应用对应 color mode

#### Scenario: 壳层右上角无主题浮层

- **WHEN** 成员处于已登录工作台任意 `/app/*`（登录页除外）页面
- **THEN** 壳层右上角不展示常驻主题切换按钮

### Requirement: 会话头不展示单聊标签且去掉活跃状态右栏

消息页单聊会话头 MUST NOT 展示「单聊」副标题或标签。消息页与通讯录页 MUST NOT 展示「活跃状态」右侧栏。若群聊成员列表原先仅在右栏，系统 MUST 提供等价入口（如会话头旁打开的成员面板），不得静默丢失。

#### Scenario: 单聊无「单聊」文案

- **WHEN** 成员打开类型为 direct 的会话
- **THEN** 会话头不显示「单聊」字样
- **AND** 群聊人数、助手等其他副标题仍可显示

#### Scenario: 无活跃状态右栏

- **WHEN** 成员打开消息页或通讯录页
- **THEN** 界面不展示标题为「活跃状态」的右侧栏

#### Scenario: 群成员仍可查看

- **WHEN** 成员打开群聊会话
- **THEN** 仍能通过会话头或等价入口查看群成员列表
